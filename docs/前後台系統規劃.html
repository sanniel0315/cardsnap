前台 × 後台
一套後端,兩個介面

CardSnap 的前台(顧客端 Web + App)負責拍照建檔與名單體驗;後台(營運管理 Admin)管理使用者、訂閱、廣告、用量與數據。兩者共用同一組後端服務與資料庫。

01 · Panorama

系統全景

三塊:前台給使用者、後台給營運團隊,中間是共用後端。前後台都只透過 API 存取後端,權限清楚分離。

02 · Front-stage

前台:顧客端

使用者實際操作的介面。核心動線是拍照 → 確認 → 建檔 → 管理 → 匯出/分享,加上登入、雲端同步與訂閱付費牆。

🧭 主要畫面與動線

入口🔐

登入 / 註冊

Google / Apple 一鍵登入;免登入也可先試用(本機暫存)。

核心📷

拍照掃描

相機取景、邊界校正、端上 OCR、欄位自動填入。

核心✅

確認 / 編輯

預填表單一眼校對,對照原圖修正後存檔。

核心🗂️

名單列表

搜尋、標籤篩選、收藏、卡片/清單檢視。

核心👤

名片詳情 / 分享

撥號、寄信、vCard、QR、Web Share。

核心📤

CSV / vCard / Excel 一鍵匯出整份名單。

Pro☁️

連結 Google Drive,多裝置即時同步。

變現💎

訂閱付費牆

觸發點顯示方案,RevenueCat 內購,7 天試用。

🧱 前台技術

03 · Back-office

後台:營運管理主控台

給內部團隊用的 Admin。看得到全平台運作、管得動使用者與訂閱、調得了廣告與功能開關,並掌握 OCR 用量成本與營收數據。

📊

營運儀表板

DAU/MAU、新增名片數、掃描成功率、訂閱轉換漏斗、MRR/營收趨勢。

👥

使用者管理

搜尋用戶、查看方案與裝置、停權/復權、重設、刪除帳號(個資請求)。

💳

訂閱與金流

訂閱狀態、試用、退款、RevenueCat 對帳、優惠碼與促銷管理。

📺

廣告管理

廣告版位開關、頻率上限、A/B 測試、AdMob 成效檢視。

🧮

OCR 用量與成本

端上 vs 雲端辨識比例、Cloud Vision 呼叫數與費用、異常用量告警。

🚩

內容檢舉 / 審核

分享連結濫用、不當內容檢舉處理、封鎖名單。

🔔

推播通知

分眾推播(免費→Pro 促購、回流提醒)、排程與成效。

🛟

客服工單

使用者回報、Intercom 整合、常見問題與回覆範本。

🎚️

Feature Flags / 設定

功能開關、灰度發佈、各方案配額(名片上限、批次)即時調整。

📜

稽核日誌

誰在何時改了什麼——所有後台操作留痕,符合法遵。

🧱 後台技術

用 Refine 或 React-Admin(或 Next.js + shadcn dashboard)快速搭建,串接後端 Admin API;圖表用 Recharts;部署在內網/受權限保護的網域,僅內部 SSO 登入。後台與前台分屬不同部署與權限域,降低風險。

04 · Backend & API

後端服務與 API

後端以 BaaS(Supabase / Firebase)為核心,搭配少量 Cloud Functions。API 分兩組:前台 API(使用者自己的資料)與 Admin API(營運、需高權限)。

🔌 前台 API(顧客端)

方法端點用途
POST/auth/loginGoogle/Apple 登入換取 token
GET/contacts取得名單(支援搜尋、標籤、分頁)
POST/contacts新增名片(欄位 + 原圖參照)
PATCH/contacts/:id編輯名片 / 收藏
DELETE/contacts/:id刪除名片
POST/ocr/enhance雲端高精度二次辨識(Pro)
POST/sync/drive備份 / 還原至 Google Drive
GET/export?fmt=csv|vcf伺服器端產生匯出檔
GET/me/subscription查目前方案 / 配額
POST/billing/webhookRevenueCat 訂閱事件(伺服器對伺服器)

🛡️ Admin API(後台,需 admin 角色)

方法端點用途
GET/admin/metrics儀表板數據(DAU、轉換、MRR)
GET/admin/users使用者清單與搜尋
PATCH/admin/users/:id停權 / 改方案 / 重設
GET/admin/subscriptions訂閱與退款管理
GET/admin/ocr-usageOCR 用量與雲端成本
PUT/admin/ads-config廣告版位與頻率設定
PUT/admin/feature-flags功能開關 / 配額
GET/admin/reports檢舉 / 審核佇列
POST/admin/push發送分眾推播
GET/admin/audit-logs稽核日誌查詢

安全分層: Admin API 走獨立網域與更嚴格驗證(內部 SSO + IP 白名單 + RBAC),前台 token 永遠拿不到 Admin 權限。資料庫以 Row Level Security 確保使用者只能讀寫自己的名片。

05 · Data Model

資料模型擴充(含後台)

在前台的 contacts / tags 之外,後台需要使用者、訂閱、廣告設定、功能開關、稽核與工單等資料表。

// ── 共用 / 前台 ──
users          { id, email, auth_provider, plan, pro_expires_at, status, created_at }
contacts       { id, owner_id→users, name, company, title, phones, emails,
                 address, website, note, image_drive_id, ocr_confidence, source,
                 is_favorite, created_at, updated_at }
tags           { id, owner_id, name, color }
contact_tags   { contact_id, tag_id }

// ── 後台 / 營運 ──
subscriptions  { id, user_id→users, store, product_id, status,
                 trial_end, renews_at, revenue, refunded }
admin_users    { id, email, role, sso_subject, last_login }   // 後台帳號
ad_units       { id, placement, enabled, frequency_cap, ab_variant }
feature_flags  { key, enabled, rollout_pct, plan_quota_json }
ocr_usage      { id, user_id, type, cost, created_at }        // 端上/雲端
reports        { id, target_type, target_id, reason, status, handled_by }
support_tickets{ id, user_id, subject, status, assignee, messages_json }
audit_logs     { id, admin_id→admin_users, action, target, before, after, at }

原則: 名片影像仍存使用者自己的 Google Drive(image_drive_id 參照),後台只看得到中繼資料與統計,看不到名片內容明細——隱私與法遵兼顧。

06 · Data Flow

關鍵資料流

三條最重要的串接,串起前台、後端與後台。

1

2

3

後台統計

掃描事件進 ocr_usage,儀表板即時更新

② 訂閱 → 解鎖

1

2

3

③ 後台調整 → 前台即時

1

2

3

前台啟動時讀 flags,無需改版即生效(灰度發佈)

07 · Access Control

後台角色權限(RBAC)

後台不是「一把鑰匙開所有門」。依職務分四種角色,最小權限原則,降低誤操作與內部風險。

👑 Owner

  • 全部權限
  • 管理 admin 帳號
  • 金流 / 退款
  • 系統設定

🛠️ Admin

  • 使用者管理
  • 訂閱檢視
  • 功能開關
  • 廣告設定

🛟 Support

  • 查使用者
  • 客服工單
  • 檢舉處理
  • 不可改金流

📊 Analyst

  • 儀表板唯讀
  • 報表匯出
  • 用量數據
  • 不可改資料

稽核: 所有後台寫入操作(改方案、退款、停權、改 flag)一律記錄於 audit_logs,含操作者、前後值與時間,可回溯。

08 · Deployment

部署與環境

前台、後台、後端各自獨立部署,三套環境(dev / staging / prod)隔離。網頁端維持「推到 GitHub 自動部署」的工作流。

🌐

前台 Web

GitHub → Pages / Vercel 自動部署;PWA 可安裝。

📱

前台 App

EAS Build 出 iOS/Android,送 App Store / Play;OTA 熱更新。

🎛️

後台 Admin

Vercel 內部網域 + SSO + IP 白名單,與前台分離。

🗄️

後端 BaaS

Supabase / Firebase 專案,分 dev/staging/prod。

⚙️

CI/CD

GitHub Actions:測試 → 部署;PR 預覽環境。

📈

監控

Sentry 當機回報、Analytics 漏斗、用量告警。
